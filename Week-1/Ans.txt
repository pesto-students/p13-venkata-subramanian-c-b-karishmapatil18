What is a protocol stack, and how is it used in web development?
ans:The protocol stack is used to allow the combination of different protocols that each set the boundaries for a number of network activities.
The most common protocol stack used in web development is the TCP/IP protocol stack.
5 layer protocol stack used in web development:
1)Application Layer:Describe how the data transfer between web browser and server.protocol used in this layer are http and https .
2)Transport Layer :Ensure Data transmission between application running on different devices.
TCP protocol used in trnsport layer.
3)Network Layer:Responsible for routing data packets across the network from the source to the destination. 
4)Data Link Layer:Responsible for maintaining link between 2 nodes of same network
5)Physical Layer:Deals with the physical transmission of data over the network medium .

Q2.What are the different types of web servers, and how do they differ in terms of functionality and performance?
Ans:web server are software application that are used to handle the http request and provide data to browser.
different types are :Apache HTTP Server,Nginx ,Microsoft Internet Information Services ,LiteSpeed Web Server,Caddy
Apache is one of the most widely used and popular web servers. It is an open-source web server known for its flexibility and robustness.
Nginx is a lightweight and high-performance web server often used as a reverse proxy server or load balancer.
IIS is a web server developed by Microsoft for Windows operating systems. It is tightly integrated with the Windows environment and is well-suited for hosting ASP.NET applications and other Microsoft technologies.
LiteSpeed is a commercial web server known for its high performance and efficiency. It claims to be faster than both Apache and Nginx in many scenarios. 
Caddy is designed to be self-contained, including features like automatic HTTP/2, HTTP/3, and QUIC support out of the box. It is ideal for developers and small-to-medium websites looking for a straightforward setup.

What is scaling, and why is it important for web applications? How does scaling differ for vertical and horizontal scaling?
Ans:Scaling Refers to the ability of an application to handle increased traffic, user load, or data without compromising its performance, responsiveness, or availability.
important for web applications for its Reliability,Performance,cost efficiency and future growth.
 Horizontal scaling involves adding more machines or nodes to a system, while vertical scaling involves adding more power (CPU, RAM, storage, etc.) to an existing machine.
